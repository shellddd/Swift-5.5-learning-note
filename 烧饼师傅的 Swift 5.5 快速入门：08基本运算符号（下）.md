## 逻辑运算符
逻辑运算符 
`a && b`
`a || b`
`!a`<br />

什么是逻辑运算符呢？用中文讲就是 **白马与神马**、 **神马或白马**、 **白马非白马** 的三个逻辑判断。<br />

例如你在我店里买面包，选了一大堆之后，你优雅地从西装口袋里摸出一张VIP会员卡，准备结账。<br />

贵——宾！您好！请输入密码：
```
let 登录密码 = false
let 支付密码 = true
if 登录密码 && 支付密码 {
    print("支付成功！")
} else {
    print("未完成支付！请重新尝试！")
}
```
![image](https://github.com/shellddd/Swift-5.5-learning-note/assets/67853933/9616fad6-904b-483a-b65a-5b049fea37be)

以上是 **白马与神马** `a && b` 的判断。逻辑运算符用2个符号 `&&` 来表示 `与`，值在其左右两侧。只有登录密码 **与** 支付密码都为 `true` 的时候，支付才会成功。如果有任意一个条件为 `false` ，支付都会失败。<br />

事实上，如果你连登录密码都为假了，那也就谈不上第二步的支付了，这就是我店为了防止VIP卡被盗用而专门布置的短路设计。<br />

你尴尬地笑了笑：唉，我这老婆，密码改了也不说一声，真是的！接着你镇静地从裤兜里掏出一部手机。<br />

先生，你好，请输入密码：
```
let 支付宝密码 = false
let 微信支付密码 = false
if 支付宝密码 || 微信支付密码 {
    print("支付成功！")
} else {
    print("未完成支付！请重新尝试！")
}
```
![image](https://github.com/shellddd/Swift-5.5-learning-note/assets/67853933/e96453ed-14d4-412a-b4c0-28bb1861a216)


以上是 **神马或白马** `a || b` 的判断，逻辑运算符用2个符号 `||` 来表示 `或`，值在其左右两侧。它表示支付宝或微信，只要有一个付款密码为 `true` ，你都可以拿着我的面包扬长而去了。<br />

但是，你又急忙地从钱包里翻出了一张信用卡。<br />

小伙子，这儿，密码：
```
let 支付密码 = false
if !支付密码 {
    print("支付成功！")
}
```
![image](https://github.com/shellddd/Swift-5.5-learning-note/assets/67853933/516fc17e-7f3f-4d23-adf8-f304ca6c57b6)


以上就是一个 ** 白马非白马**  `!a` 的判断。 逻辑运算符用英文感叹号 `！` 来表示 `非` ，值紧跟在它后面，没有空格，它的作用就像是公孙龙跟一个布尔值唱反调，你左我就右，你真我就假。所以上面的逻辑是，你的信用卡密码输入错误 `false` ,然后你告诉我，假如——你没有 `!` 输入错误，那就代表支付成功了。<br />

##组合逻辑运算符
我们可以组合多个逻辑运算符来创建更长的复合条件表达式：
```
if 登录密码 && 支付密码 || 支付宝密码 || 微信支付密码 || 信用卡密码 {
    print("支付成功！")
} else {
    print("未完成支付！请重新尝试！")
}
```
这个意思就是，你一次把会员卡、手机、信用卡都撂桌上了，一个个挨着试，哪个对就结哪个。
>注意，逻辑运算符是从左至右先后开始计算的。

##显式括号
`()` 是个好东西，在一些复杂的表达式当中，合理运用括号，可以让我们的代码更好看，更容易阅读，例如上面的组合代码：
```
if (登录密码 && 支付密码) || 支付宝密码 || 微信支付密码 || 信用卡密码 {
    print("支付成功！")
} else {
    print("未完成支付！请重新尝试！")
}
```
不是一家人，不进一家门，谁跟谁是一家，这下一眼就瞅清楚了吧。

##三元条件运算符
三元条件运算符，其实就是条件语句的简写方式，这是 Swift 中一个特殊的运算符，它具有3个部分，表现形式为：
`条件 ? 真 : 假`
如果条件成立，返回真的值，如果条件不成立，就返回假的值。我们先用条件语句来书写一次：
```
var 那小子付钱没:Int = 0
var 收银款:Int
if 那小子付钱没 != 0 {
    收银款 = 0
} else {
    收银款 = 8999
}
print(收银款)
```
这个代码什么意思呢？就是说，那小子付没付钱，其实我也不清楚，但是我感觉他没付，所以我给它赋值为 `0` ，假如 `那小子付钱没` 不等于 `0` ，就说明这个条件不成立，那人就付钱了。
![image](https://github.com/shellddd/Swift-5.5-learning-note/assets/67853933/eebaca5f-f722-482b-8f3d-6a427b1f87ab)

接下来，看你三元哥的表演：
```
var 这小子付了钱:Int? = 8999
收银款 = (这小子付了钱 != nil) ? 8999 : 0
print(收银款)
```
这又什么意思呢？三元哥认为这小子付钱与没付钱，各占一半，所以他用了一个可选值为 `8999` 和空值 `nil` ，还记得前面讲过的可选值类型吗？假如 `这小子付了钱` 不等于 `nil` ，就说明条件成立，那就返回 `:` 左边的 真 。
![image](https://github.com/shellddd/Swift-5.5-learning-note/assets/67853933/3d4cff90-046d-431e-894d-bb005ca5e58e)

怎么样？一支代码就搞定，有没有很简洁！
## 空合运算符
**空合运算符** 
`a ?? b` 
这是专门针对可选值类型 `a` 进行是否为空的判断运算符。注意，这个可选值只能在 `??` 的左边。如果 `a` 中有值，就解开包裹看，里面有啥，就返回啥，要是为空就返回 `??` 右边的元素。
下面看你空爷露一手：
```
var 那孙子付钱没:Int? //如果不赋值，则默认为 nil。
收银款 = 那孙子付钱没 ?? 0
print(收银款)
```
空爷是老江湖了，搭眼一瞧，就知道你这怎么回事了，门儿清。所以他给了一个为空的可选值。
